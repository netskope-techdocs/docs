      Incidents Endpoint Schema

      Parameter

      Type

      Description

      access_method

      str

      This field shows the actual access method that triggered the event.

      acting_user

      str

      Acting User is the user responsible for the violation.

      activity

      str

      This field shows the user activity that triggered the event.

      app

      str

      Specific app name used by the enduser.

      app_session_id

      int

      app_session_id to check all the user activities in a single app session. app_session_id is unique for a user, device, browser and domain.

      assignee

      str

      Represents the username to whom the incident is assigned to.

      bcc

      str

      BCC users information in the case of SMTP DLP incidents

      cc

      str

      CC users information in the case of SMTP DLP incidents

      channel

      str

      Channel name in Messenger applications eg. Microsoft Teams, Slack for Enterprise

      classification

      str

      Classification of the incident.

      connection_id

      int

      A connection ID is a unique integer identifier that is assigned to each connection

      dlp_file

      str

      Original DLP file name.

      dlp_incident_id

      int

      DLP incident ID

      dlp_match_info

      list

      DLP match info carries the details about DLP profile and rule info along with the violaton count information.

      dlp_parent_id

      int

      DLP parent incident ID infromation.

      dst_location

      str

      Application's city as determined by the Maxmind or IP2Location Geodatabase

      exposure

      str

      The exposure information of the incident ( for ex Public , Private etc ).

      file_lang

      str

      The language of the file content. ( ex ENGLISH)

      file_path

      str

      File path of the object that triggered the incident.

      file_size

      int

      File size of the object that triggered the incident.

      file_type

      str

      File type of the object that triggered the incident.

      from_user

      str

      From user information in the case of SMTP DLP incidents

      _id

      str

      Unique hexadecimal string representation of the incident object

      inline_dlp_match_info

      list

      Inline DLP match info carries the details about DLP profile and rule info along with the violaton count information.

      instance

      str

      The cloud app instance name associated with the incident.

      instance_id

      str

      Instance Id of the incident specific to the app.

      latest_incident_id

      int

      Latest incident ID

      md5

      str

      MD5 value of the file content.

      object

      str

      Incident object name and the value of the field represents the object details of the incident triggered.

      object_type

      str

      Type of Object with dlp violations eg. File, Mail, Body, Extension,Form, Image, Text, message, attachment etc.

      owner

      str

      The file owner information.

      owner_pdl

      str

      Region

      referer

      src

      Referer URL of the application(with http) that the user visited as provided by the log or data plane traffic

      severity

      str

      Severity of the incident

      site

      str

      Specific site name used by the enduser.

      src_location

      str

      Application's city as determined by the Maxmind or IP2Location Geodatabase

      status

      str

      Incident status ( ex New , Resolved etc )

      timestamp

      str

      Epoch timestamp when the incident got triggered.

      title

      str

      Title of the DLP incident

      to_user

      str

      To users information in the case of SMTP DLP incidents

      true_obj_category

      str

      Actual object category ( ex Text )

      true_obj_type

      str

      Actual object type ( ex Plain Text file or Excel )

      url

      str

      URL of the object with dlp violations

      user

      str

      User who triggered this incident.
