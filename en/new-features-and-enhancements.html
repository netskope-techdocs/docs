New Features and Enhancements

Category

Feature

Detailed Description and Benefits

Advanced Analytics

New dimension added for Network events

With this release, we have added a new "publisher_cn" field to Network events.

This field contains the Common Name of Netskope Private Access Publisher associated with the event.

Advanced Analytics

New dimension added for Netskope data centers

With this release, we have added a new dimension "netskope_pop_location".

This field provides the location of the Netskope data center through which the traffic was routed.

API Protection

Support for ServiceNow Quebec release

Netskope has now validated support for ServiceNow Quebec release for Netskope API Data Protection.

CASB Inline Protection

DLP support for Google Calendar invites

With this release, we have added DLP support for Create, Edit, and Send activities via Google calendar. Users can send confidential information via the description field.

Consider the following ways to add / edit the description field:

  • When creating a new calendar, add text to the description field and ensure you have a DLP policy in place and verified.

  • Click the calendar edit button and update the description field.

  • Click calendar events, click the email guest buttons and update the description field.

CASB Inline Protection

Planview LeanKit enhancement

Traffic for the Leankit application is enhanced to cover the following activities:

  • Create

  • Edit

  • Delete

  • Post

  • Upload

  • Download

  • Share

  • Move

  • Login Attempt

  • Login Failed

  • Login Successful

  • DLP is supported for Upload and Download

Directory Services

Netskope Adapters

The Netskope Adapter (NS Adapter) has been tested to ensure compatibility with the current cloud platform. Its version number has been updated to confirm this compatibility. No other changes have been made to the NS Adapter in this release.

SSPM

SkopeIT Alerts name changes for SSPM

On the SkopeIT Alerts page, each alert's details now include the following name changes:

  • "Iaas Asset Tags" has been moved to the Resource section and renamed to "Asset Tags"

  • "Sa Profile Name" has been renamed to "Profile Name"

  • "Sa Rule Severity" has been renamed to "Rule Severity"

  • "Sa Rule Name" has been renamed to "Rule Name"

  • "Sa Rule Remediation" has been renamed to "Remediation"

  • "Ccl" has been renamed to "CCL"

  • "Iaas Remediated" has been renamed to "Remediated"

Steering

Client enhancement to use all external / static configured proxies to intercept traffic

The Netskope Client now sets interception filters for all configured proxy interoperation destinations set in the steering configuration.

Successful health check of the configured proxies is no longer required.

Threat Protection

Standard / Basic Threat Protection license updated

This feature enables new actions from the Malware page for detected malware files as follows:

  • Quarantine - Applies only for API enabled protection when this action is detected. Allows quarantine action with quarantine profile selection and confirmation. This action is asynchronous and the action field will reflect the new state upon task completion and is logged.

  • Restore - Applies only for API enabled protection for files that have been quarantined using the quarantine action. Allows a quarantined file to be restored. This action is asynchronous and the action field will reflect the new state after task completion and is logged.

  • Download - Applies only for API enabled protection. Allows a detected malware file sample to be downloaded as a password protected zip file. The password is the same as the setup in Settings > Threat Protection > API enabled protection.

  • Mark Safe - Applies to all malware files. Allows providing feedback to Netskope threat research that the file was deemed safe for the account. An additional link "add to file hash list" is provided in the notification. Using this link, the file can be added to the selected (one or more) file hash list(s) for Allow list.
