New Features and Enhancements

Category

Feature

Detailed Description and Benefits

API Protection

Malware Support for Slack Enterprise

API-enabled Protection now supports malware detection and quarantine for Slack Enterprise app.

Important

  • Malware detection is not supported for chat messages and snippets.

  • You cannot use the Slack Enterprise quarantine profile as the destination for malware detected files.

However, if you have set up any API-enabled Protection-supported storage apps, you can leverage the quarantine profile of these apps.

API Protection

Microsoft Office 365 OneDrive and SharePoint

API-enabled Protection now supports scanning files within predefined Office 365 enterprise groups, for example, Everyone and Everyone except external users.

API Protection

Box Centralized Ownership

This feature applies to a Box account which has a centralized ownership model where an administrator creates individual folders for each user within an administrator's home folder and shares such folders with respective users.

To mitigate the chances of hitting API rate limits in operations such as file-listings for such account configurations, contact Netskope support to enable this feature for your account.

CASB Inline Protection

API research and enhancements for GitLab

Added support for Upload and Download activities for GitLab performed via APIs.

CASB Inlinne Protection

Instance ID support via synthetic requests

We have added Instance ID support to the CLI activities for AWS S3, EC2, SNS, Lambda and RDS services.

Directory Services

Netskope Adapters

The Netskope Adapter (NS Adapter) has been tested to ensure compatibility with the current cloud platform. Its version number has been updated to confirm this compatibility. No other changes have been made to the NS Adapter in this release.

NG SWG / CASB

Support RFC compliant XAUHeaders in Forward to Proxy profiles

With this release, we now support RFC compliant X-Authenticated-Userheader in the Forward to Proxy feature.

Netskope for IaaS

Compliance standards fields supported in Alerts UI

Security Assessment Alerts change to provide new and better formatted Compliance Standards information across table, Alert Details, and Export.

Steering

Netskope Client support

The Netskope Client is qualified on macOS M1 11.2.

Steering

Failclose implemented for parameters installation

The following options are implemented for command line parameters installation.

  • fail-close=disable This explicitly disables failclose.

  • fail-close=no-npa This excludes NPA traffic when fail closed.

Steering

Netskope Client certificate

For stronger security, the Netskope Client user cert can no longer be replaced with other user certificates.

UEBA

All nine UBA rule-based predefined policies and incidents visible for all admins

User Behavior Analytics : Rule based pre-defined policies and related incidents are now available on the Behavior analytics page.

These Behavior Analytics policies will replace the policies on the anomalies page.

If a policy is enabled on both the anomalies page and the behavior analytics page (for migration reasons), you will see duplicate alerts corresponding to each detection.

UEBA

Shared credentials for UBA predefined policies

Shared Credentials is now available as a pre-defined policy under Behavior Analytics.

UEBA

Deprecating the Anomalies page

The anomalies page will soon be deprecated and customers should use the Behavior Analytics page to enable corresponding policies.

If a policy is enabled on both the anomalies page and the behavior analytics page (for migration reasons), you will see duplicate alerts corresponding to each detection.

Web UI

Renaming Incident Management to Forensics

Incident Management in Settings has been renamed to Forensics to more accurately convey what's on the page.
