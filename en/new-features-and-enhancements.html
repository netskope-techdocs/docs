New Features and Enhancements

Category

Feature

Detailed Description and Benefits

Behavior Analytics

REST API export UCI scores

With this release, the UCI score API is available for admins to retrieve a user's confidence score over a period of time.

Contact Support to enable this feature in your account.

CASB Inline Protection

Yandex app suite enhancement

Activities for the Yandex apps are detected under the app name 'Yandex App Suite'. A new app connector has been added to support the following activities:

  • Login Attempt

  • Login Successful

  • Login Failed

  • Logout

The Yandex App Suite has multiple apps like Yandex.Disk and Yandex Mail. Previously, the log in for Yandex apps were detected with the app name 'Yandex.Disk'.

CASB Inline Protection

Connector development and automation

Inline CASB for AWS is enhanced to provide broader service coverage and enrichment of inline events with the account ID.

CASB Inline Protection

JIRA integration new coverage for Google Drive and Dropbox

With this release, JIRA integration is enhanced with added coverage for Google Drive and Dropbox. Admins can attach file links and remove links for the file from cloud storage.

Data Protection

New Entity UI for all accounts

The New Entity UI was previously enabled by default for all new accounts only. However, with this release it is now enabled for all accounts.

The Entity UI is used for selecting entities when creating new DLP rules.

New Entity UI: The New Entity UI allows you to navigate the entire set of entities easily as they are now organized into an intuitive tree structure with eight (8) levels on the top. In addition to the ability for you to easily browse, we are also introducing an enhanced search capability.

All entities have been updated to have a user-friendly display name, a succinct description, and sample data that will match the entity.

Also included with the New Entity UI are new Source Code entities.

Source Code Entities: The new Source Code entities are designed to provide more accurate detection of files containing source code or snippets of source code in a variety of programming languages.

Once the New Entity UI is enabled, the old Source Code entities will no longer be available for use in any custom DLP rules. Any existing custom DLP rules will continue to work but no edits to those rules will be allowed.

It is recommended that customers switch to using the new Source Code entities. In order to do so, any custom rules containing the old Source Code entities/data identifiers will need to be deleted and new custom rules will need to be created using the new Source Code entities.

Directory Services

Netskope Adapters

The Netskope Adapter (NS Adapter) has been tested to ensure compatibility with the current cloud platform. Its version number has been updated to confirm this compatibility. No other changes have been made to the NS Adapter in this release.

Netskope for IaaS

GCP CIS 1.2 update

CIS Google Cloud Platform Foundation Benchmark version 1.2.0 is now part of the Security Posture predefined profiles.

This is the latest version of GCP CIS benchmarks.

Netskope for IaaS

AWS CIS 1.4 update

Security Posture now includes a predefined profile for the CIS Amazon Web Services Foundations v1.4.0 benchmark.

Netskope for IaaS

Security Assessment for AWS enhancements

Continuous Security Assessment for AWS now supports the following enhancements:

  • Support listing of new AWS services: Customer Gateway, VPN Gateway, Elastic IP, CloudFront Distribution

  • VPC now supports "IsDefault" metadata

  • S3Bucket now supports "BucketLifeCycle Policy" metadata

  • KMSKey now supports "KeyManager" metadata

  • RDS now supports "LatestRestorableTime" and "KMSKey" reference

Netskope for IaaS

Compliance Findings/Resource and Alerts enhancement

With this release, we have added Resource ID and Resource Group to Compliance, Alerts, and Client API.

File size constraint enhancement

Steering

Support for domain name exception for UDP traffic

Netskope client now supports bypassing the UDP traffic for hostname or IP address based exceptions. However, domain name exception for UDP traffic will not work on Big Sur.

Contact Support to enable this feature in your account.

Steering

Netskope client improved security

The Netskope client implemented the self protection feature which prevents the termination of the Netskope service and UI processes by administrative privileged users. The self protection also prevents the modification of the config files and the Netskope specific registry entries.

This feature is supported in Windows 10+ OS only. It is not supported on Windows 7/8, Windows 10, ARM64, and macOS.

Contact Support to enable this feature in your account.

Web UI

Widget library UI enhancements

The widget library provides new improved instructional text in the UI and cleaned up spacing between components and text.
