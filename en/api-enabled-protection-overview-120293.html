                                                                                                                                                                                                                                                                                                                                                                                                                            API-enabled Protection Overview

                                                                                                                                                                                                                                                                                                                                                                                                                            Use an out-of-band API connection into your sanctioned cloud services to find sensitive content, enforce near real-time policy controls, and quarantine malware. This deployment option has the advantage of being simple and friction-less to deploy, requiring only a few steps and granting access to the sanctioned app from the Netskope console using OAuth. The other advantage is that the API connection enables inspection of content that already resides in the sanctioned app. This is not possible with a proxy deployment. There are two limitations to API API-enabled Protection. First, visibility and control is out-of-band, so visibility and control are after-the fact versus proactive and real-time. Second, only sanctioned cloud services are supported.

                                                                                                                                                                                                                                                                                                                                                                                                                            Using API-enabled Protection involves first configuring the API connection for a sanctioned app, and then creating API-enabled Protection policies that scan for policy violations. Lastly, use the API-enabled Protection dashboard to view analytics about your sanctioned app. API-enabled Protection currently supports Amazon Web Services, Azure, Box, Cisco Webex, Dropbox for Business, Egnyte, GitHub, Gmail, Google Drive, Google Cloud Platform, OneDrive, Outlook, SharePoint, Microsoft Teams, Salesforce, ServiceNow, Slack Team, Slack Enterprise, and Workplace by Facebook.
