Create a Real-time Protection Policy for CTEP

After creating an CTEP profile and enabling it, create a policy to use the CTEP profile. Go to Policies > Real-time Protection.

Note

These instructions are for the new Real-time Protection policy workflow. If you have not yet opted-in to this feature, do so before proceeding.

  1. On the Real-time Protection Policies page, click New Policy and select CTEP.

  2. For Source, click in the text box to select users, user groups, or organizational units. Using the default User = Any is recommended. To add criteria, like Source IP, Access Method, etc., select one or more options from the dropdown list.

  3. For Destination, click in the text box to select Category. Specify a web category or entire groups of web categories, to allow traffic in order to prevent an exploitation. You can also add criteria for Destination.

  4. For Profile and Action, select the CTEP profile you created (or use a default profile). Leave CTEP as the Action.

  5. For Set Policy, enter a name for this policy.

    To enter a description, click + Policy Description. To send email notifications, click + Email Notification, select how often to send the email, specify who should receive it, and then click Done.

  6. When finished, click Save in the top right corner of the page.

Important

On the Real-time Protection policies page, certain types of policies should be in a particular order. Place your CTEP policies at the bottom of the table.

To move policies on the page, click the icon for a policy, and then select Move to change the order of policies on the page.

To see any policy violations detected by CTEP, click View Alerts to go to the SkopeIT Alerts page.
