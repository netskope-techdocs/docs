Configure the API Connector for Cisco Webex Teams

To use API-enabled Protection with Cisco Webex Teams, first you need to connect Netskope to your Cisco Webex Teams app with APIs. The following sections explain how the API connectors work and how to configure the connection.

API-enabled Protection API Connectors

Netskope API-enabled Protection works by directly connecting to the cloud app using the APIs published by the app, and uses OAuth to gain delegated access to the app.

Netskope's API-enabled Protection provides a complementary deployment model to provide cloud visibility, policy, and data security services by directly connecting to the cloud service using the APIs published by the cloud services. The API Connector works in conjunction with the Netskope cloud proxy to provide defense-in-depth security services. The API Connector is only available for data security related services such as DLP, Active Encryption, and Active Permissions Management.

Configure Cisco Webex Teams for API-enabled Protection

To configure Cisco Webex Teams for API-enabled Protection, you need to authorize users in Cisco Webex Teams and grant access a in your Cisco Webex Teams instance.

There are two parts to this procedure:

  • Configure Cisco Webex Teams user privileges

  • Configure Cisco Webex Teams instance in the Netskope UI

Configure Cisco Webex Teams User Privileges

Use your Webex Teams admin account to add user privileges and obtain the email address used in the Netskope app.

To configure Cisco Webex Teams for Netskope:

  1. Log in to your Webex Teams admin account, and clickÂ Manage Users.

  2. Add a user or use an existing user to grant access to the Netskope app.

  3. On the Users page, select the user and enableÂ the Full administrator and Compliance Officer privileges.

  4. Save the changes, and then copy the email address for this user. which is needed to add the Netskope instance in the next section.

Configure Cisco Webex Teams Instance in the Netskope UI

To authorize Netskope to access your Webex Teams instance:

  1. Log in to the Netskope tenant UI: https://<tenant hostname>.goskope.com and go to Settings > API-enabled Protection > SaaS.

  2. Select the Cisco Webex Teams icon, and then click Setup Instance.

  3. The Setup Instance window opens. Enter the name for your Webex Teams account, make sure the API-enabled Protection check box is enabled, and then enter the email address for the user that has full administration and compliance officer privileges.Â Also enter the organization name (case-sensitive) copied from the Cisco Webex teams admin portal.

  4. Click Save, then click Grant Access for the app instance you just created. You will be prompted to log in with your admin username and password, and then click Grant Access. When the configuration results page open, click Close.

Refresh your browser, and you should see a green check icon next to the instance name.
