Operations

Only a Super Admin with SSH access to the server on which the CE platform is installed can perform these operations. This section provides useful information for the Admin who manage the CE infrastructure.

Reset Password

To reset the Super Admin password, perform the steps on the machine that is running the docker containers.

  1. Enter the mongodb docker container:

    docker exec -ti mongodb sh

  2. Login to the mongodb client. Enter the value of the MONGO_PASSWORD environment variable (from docker-compose.yml) when prompted for a password.

    mongo --username cteadmin

    Enter password:********

  3. Switch to the CTE database:

    use cte

  4. Reset the Super Admin password:

    db.users.update({username: "admin"}, {$set: {password: "$2y$12$RBcV6xWFhHucm4a1YRmQXuEZHqz9NadpMuzIB6xEIXOhg.QzngiiO"}});

  5. The Super Admin password has now been reset to default.

  6. Login to the Netskope CE UI with default password and change the password of your choice.

Log Cleanup

To remove log files perform the steps shown below on the machine that is running the docker containers.

  1. Enter the core docker container:

    docker exec -ti core sh

  2. Remove log files:

    rm -f /logs/celery /logs/celery-beat /logs/gunicorn /logs/rabbitmq

  3. Restart the core container:

    docker-compose restart core

To remove logs from the database perform the steps shown below on the machine that is running the docker containers.

  1. Enter the mongodb docker container:

    docker exec -ti mongodb sh

  2. Login to the mongodb client. Enter the value of the MONGO_PASSWORD environment variable (from docker-compose.yml) when prompted for a password.

    mongo --username cteadmin

    Enter password: *******

  3. Switch to the CTE database:

    use cte

  4. Remove the logs:

    db.logs.remove({})

Note

The CE platform doesn't support auto log rotation. This feature may be addressed in future releases.

Update Plugin Configuration Checkpoint (Last Run time)

To fetch historical data for an existing configuration, the checkpoint has to be manually updated.

To do this, go to Configurations and edit the configuration you want to modify. Update the Last Run value to an appropriate historical date-time and save the configuration. The next time the configuration runs, data will be fetched from that newly entered date-time.
