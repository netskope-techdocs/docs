Configure GitHub for API-enabled Protection

To configure GitHub for API-enabled Protection, you need to authorize Netskope as a web application client to access your GitHub instance. 

To configure GitHub for API-enabled Protection, there are two parts to this procedure:

  • Install the API-enabled Protection for GitHub app in the GitHub organization account

  • Configure GitHub instance in the Netskope UI

Install API-enabled Protection for GitHub App in GitHub

To install the API-enabled Protection for GitHub app in the GitHub organization:

  1. Log in to www.github.com using your GitHub organization account.

  2. Based on the location of your Netskope tenant, install the API-enabled Protection for GitHub app from the following URLs:

    Note

    Ensure that you are logged in to your GitHub organization account before accessing the following URLs.

    • https://github.com/apps/introspection-for-github-de (for Netskope tenants based out of FR4 management plane)

    • https://github.com/apps/introspection-for-github-eu (for Netskope tenants based out of AM2 management plane)

    • https://github.com/apps/introspection-for-github-na (for Netskope tenants based out of SV5 management plane)

    • https://github.com/apps/introspection-for-github-na-sjc (for Netskope tenants based out of SJC1 management plane)

    Note

    If you are not sure of the management plane location of your tenant, contact your sales representative or Netskope support.

  3. Click Install.

  4. Select the organization name and click Install.

    Note

    Keep the installation options unchanged.

Once installed, you should see a successful message. Proceed to configure the GitHub instance in Netskope UI.

You can uninstall the API-enabled Protection for GitHub app too. To uninstall the app:

Note

You can skip this procedure if you continue to use the API-enabled Protection for GitHub app.

  1. Log in to www.github.com using your GitHub administration username.

  2. On left of the page, select your organization name from the drop-down list.

  3. On the top-right, click View organization.

  4. Click the Settings tab.

  5. On the left navigate pane, click Installed GitHub Apps.

  6. Identify the API-enabled Protection for GitHub app and click Configure.

  7. Scroll down to the Uninstall <app name> section and click Uninstall.

Once you uninstall the app, Netskope does not receive any notifications from GitHub.

Configure GitHub Instance in Netskope UI

To authorize Netskope to access your GitHub instance:

  1. Log in to the Netskope tenant UI: https://<tenant hostname>.goskope.com and go to Settings > API-enabled Protection > SaaS.

  2. Select the GitHub icon, and then click Setup Instance.

  3. The Setup Instance window opens. Enter the following details:

    • Instance Name: Enter the name of the GitHub organization.

      Note

      The instance name must be same as the GitHub organization name. It is case-sensitive.

    • Instance Type: Select the appropriate instance type from the following options:

      • API-enabled Protection  

      • Audit Reports

    • Admin Username: Enter GitHub username of the administrator.

      Note

      The username must be same as the GitHub username of the administrator that you used to install the API-enabled Protection for GitHub app. It is case-sensitive.

  4. Click Save, then click Grant Access for the app instance you just created. You will be prompted to log in with your admin username and password, and then click Grant Access. When the configuration results page opens, click Close.

Refresh your browser, and you should see a green check icon next to the instance name.
