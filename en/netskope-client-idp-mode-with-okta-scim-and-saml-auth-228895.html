Netskope Client IdP Mode with Okta SCIM and SAML Auth

You can deploy the Netskope Client in IdP Mode specifically for Windows and MAC endpoints that are not joined to a domain and use a Cloud Identity and Access Management (like Okta). For other deployment modes, refer to the Netskope Support Knowledge Hub.

These instructions apply to both types of tenant (Cloud Apps Only or Web Traffic).

Workflow

This integration requires the following steps to be performed in the order specified.

  1. Configure Netskope OAuth Token for Okta SCIM Integration

  2. Configure Okta SCIM Integration to Onboard Users to Netskope

  3. Get Netskope SAML settings

  5. Add an Okta IdP Account in Netskope SAML â Forward Proxy

  6. Enable Authentication and Configure Domain Bypass Settings

  7. Install the Netskope Client in IdP Mode and Enroll Users For Windows

  8. Install the Netskope Client in IdP Mode and Enroll Users For Mac

  9. Test Website Access and Check SkopeIT Events
