Deploy Netskope Client with JAMF

JAMF is an enterprise mobility management tool that is used for the endpoint management of macOS devices. You can install the Client on users' device using JAMF ( formerly known as Casper Suite ).

JAMF Deployment Prerequisites

  • Administrators must possess fair knowledge of JAMF/JSS/Casper suite.

  • This procedure provides JAMF/JSS configuration instructions for the clients installed in AD controlled macOS devices.

  • Download the JAMF scripts from the Download page in Netskope Support portal. The file contains the essential command-line executable script to install and configure the client. The script file is available from the Netskope support portal.Â 

User Configuration

Execute the script to download the configuration file. This script locates active (online) AD users and downloads user specific configuration files from the Netskope cloud to the end point. Ensure that the AD devices are accessible before executing the script.

Installing the Client

Client installation is done using JAMF policies. The following section describes in detail on creating JAMF policies.

Create a New JAMF Policy

  1. In the JSS Dashboard, go to Computer > Policies and click + New.

  2. On the General page, enter a Display Name, for example: Netskope Client Policy.

  3. For Trigger, select Login. Scripts can also be run using other options, like Logout and Network State Change.

  4. For Execution Frequency, select Once per computer.

  5. Select Packages and on the Packages page, click Configure.

  6. Add the Client installer package, and for Action, select Install.

  7. Select Scripts and on the Scripts page, add the jamfnsclientconfig_<version-number>.sh script.

    • For Priority, selectÂ Before. The script must be executed before the installation process, so Priority must be Before.

    • Netskope supports five modes of deployment. Update the script options: Parameters 4 to 8 as follows for each mode:

      • Parameter 4: Value = IDP. This parameter specifies that client installation is being done in IdP Provisioning mode.

      • Parameter 5: Domain name. For example: if your tenant URL is https://corp.example.com, enter example.com

      • Parameter 6: Tenant name. For example: if your tenant URL is https://corp.example.com, enter corp

      • Parameter 7: To request email settings. Enter 0, if you do not want to request email ID from user. Enter 1, to request email ID from user.

      Standard Mode (email-based)

      • Parameter 4: Your tenant name.

        If your tenant URL is https://<tenant hostname>.goskope.com, then enter <tenant hostname>.goskope.com.

      • Parameter 5: Your AD name.

      • Parameter 6: Your REST API token.

        In your tenant (Netskope admin console), go to Settings > Tools > REST API to get this token. If you are generating your token for the first time, click the Generate Token button.

      UPN Mode

      • Parameter 4: Your addon URL.

        If your tenant URL is https://<tenant hostname>.goskope.com, then enter addon-<tenant hostname>.goskope.com.

      • Parameter 5: Your Organization ID.

      • Parameter 6: Enter the keyword upn.

      Multi-user Mode (enabling for each provisioned user on the tenant)

      • Parameter 4: Your addon URL.

        If your tenant URL is https://<tenant hostname>.goskope.com, then enter addon-<tenant hostname>.goskope.com..

      • Parameter 5: Your Organization ID

      • Parameter 6: Enter the keyword peruserconfig.

      IDP Single-User mode

      • Parameter 4: Enter IDP to specify the client deployment mode is IDP.

      • Parameter 5: Domain name. Example, if your tenant URL is https://corp.example.com, then enter example.com

      • Parameter 6: Tenant name. Example: If your tenant URL is https://corp.example.com, enter corp.

      • Parameter 7: Email Address request option. Enter 0, if you do not want request user's email address. Enter 1 to request user's email address.

      IDP Multi-User mode

      • Parameter 4: Enter IDP to specify that the client deployment is in IDP mode.

      • Parameter 5: Domain name. Example, if your tenant URL is https://corp.example.com, then enter example.com

      • Parameter 6: Tenant name. Example: If your tenant URL is https://corp.example.com, enter corp.

      • Parameter 7: Email Address request option. Enter 0, if you do not want request user's email address. Enter 1 to request user's email address.

      • Parameter 8: Enter peruserconfig to specify multi-user IDP deployment mode.

      For macOS devices that are not AD joined

      Note

      • Supported only for non-AD joined and single-user installations.

      • Parameter 4 : Your tenant URL

      • Parameter 5 : Your Rest API token.

        In your tenant (Netskope admin console), go toÂ SettingsÂ >Â ToolsÂ >Â REST APIÂ to get this token. If you are generating your token for the first time, click theÂ Generate TokenÂ button.

      • Parameter 6 : Preferences file (plist)Â  name.

        Note

        When entering the filename, enter the complete filename including the extension. Example: netskope.plist and do not add HTTP: to the URL in the plist file.

      • Parameter 7 : Enter the keyword preference_email.

  8. Click the + button to add another script.

  9. When finished, click Save.

Configuration Profile for Auto Approval of Network Extension

Login to JAMf with admin credentials.

  1. Go to Computers > Configuration Profiles > Options > System Extension

  2. Select Allow users to approve system extensions

  3. Add Network Extension Team ID. Team Identifier: 24W52P9M7W

  4. Under System Extension Types, > Allowed System Extensions select com.netskope.client.Netskope-Client.NetskopeClientMacAppProxy

  5. Optionally, if older clients are used in the same profile, you can combine kernel extensions.

    1. Go to Approved Kernel Extensions

    2. Select Allow users to approve kernel extensions and FIELD_ALLOW_NON_ADMIN_USER_APPROVALS.

    3. Create extension bundles by adding KERNEL EXTENSION BUNDLE ID.

Configuation Profile for Auto Approval of VPN Payload

  1. Go to Computers > Configuration Profiles > General

  2. Click the Edit button. Under the Options section, select VPN and configure the following:

    1. VPN Type : Select Per-App VPN

    2. Per-App VPN Connection Type: Select Custom SSL

    3. Identifier: Enter : com.netskope.client.Netskope-Client

    4. Server: Enter Netskope Gateway.

    5. Provider Bundle Identifier: Enter com.netskope.client.Netskope-Client

    6. Provider Type: Select App-Proxy

    7. Select Include All Networks

    8. Specify Provider Designated Requirement: Enter anchor apple generic and identifier "com.netskope.client.Netskope-Client" and (certificate leaf[field.1.2.840.113635.100.6.1.9] /* exists */ or certificate 1[field.1.2.840.113635.100.6.2.6] /* exists */ and certificate leaf[field.1.2.840.113635.100.6.1.13] /* exists */ and certificate leaf[subject.OU] = "24W52P9M7W")

    9. Select Prohibit users from disabling on-demand VPN settings

Push Netskope Root and Tenant Certificates

Before you can push the root and tenant certificates, ensure that you do the following:

  1. Download root and tenant certificates from Netskope MDM distribution page.

    1. Login to Netskope WebUI with admin credentials.

    2. Go to Settings > Security Cloud Platform > MDM Distribution. The certificate download options are displayed in the Certificate Setup section.

  2. Convert the downloaded certificates to .cer format.

To push certificate via JAMF, login to JAMF admin console and do the following:

  1. Go to Computer > Configuration Profile > New.

    1. Under Options, give a name to this profile.

    2. Select Certificate > Configure

  2. Enter a name for the certificates.

  3. Select Upload to upload the converted root and tenant certificates.

  4. In the Scope tab, select the target computers.

  5. Click the Save button.

Verify Client Installation

Check the installation logs on the user's machine in the /var/log/install.log folder. If the user configuration download script fails and the Netskope client installer is executed, the installer will exit and displays the "Configuration file missing, aborting installation! errorâ message.

Check Netskope Client Installation Status

  1. To verify the status of each device, go to Computer > Policies and click on the policy you created.

  2. Click the Logs button at the bottom to view the log files for each device and then click the Show button.
