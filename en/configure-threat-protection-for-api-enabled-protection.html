Configure Threat Protection for API-enabled Protection

Netskope can scan files stored in your cloud storage applications for malware. To scan files for malware using API-enabled Protection, first you need to:

  • Enable the malware option in an API-enabled Protection app instance. For example, go to Settings > API-enabled Protection Instances, and select an existing app instance. In the Accounts Details dialog box, enable the malware (scanning) checkbox. Do this for all app instances for which you want to apply a malware scan.

  • Create a Remediation Profile.

To configure threat protection for API-enabled Protection:

  1. In the Netskope tenant UI, go to Settings > Threat Protection > API-enabled Protection.

  2. Enable the app instances you want to scan by enabling the toggle beside the instance name.

  3. Under Settings, review how your SkopeIT alerts, quarantine, and remediation profiles are set up. Green check-marks show which of these settings are enabled.

  4. Click EDIT SETTINGS to configure your malware settings. By default, malware remediation profiles are turned off.

  5. Configure these settings to activate threat protection for low, medium, and high severity levels:

    1. Choose the action you want to take for each severity level. Select to create a SkopeIT alert or quarantine file, which zips the password-protected file and notifies the admins specified.

      Tip

      For low and medium severity malware scans, SkopeIT alerts are recommended. For high severity scans, quarantine is recommended.

    2. Enable the Remediation Endpoints checkbox to use a remediation profile during the scan.

    3. Select the quarantine and remediation profile to use for scanning files.

  6. Add a password for file that get zipped.

  7. Select whom to notify when files are flagged for quarantine and remediation.

  8. When finished, click Save.

Now you are ready to use the malware and malicious sites pages.
